# سجل بيانات برامج خدمة المجتمع — البنك الأهلي السعودي (SNB)

نموذج ويب عربي (RTL) لجمع بيانات السجلات الخاصة ببرامج المسؤولية المجتمعية («أهالينا»)،
مع حفظ تلقائي ومتابعة على مراحل وتصدير إلى Excel. **لا يحتاج خادماً — استضافة ثابتة بالكامل.**

**الرابط المباشر:** https://baraab-bit.github.io/al-ahli-sijillat/

---

## كيف يستخدمه العميل

1. يفتح الرابط في المتصفح (الجوال أو الحاسب).
2. يتنقّل بين البرامج السبعة من الشريط العلوي الثابت (الإنتاج الحرفي · الإسكان · المبادرات البيئية · التطوع · التمويل المصغر · ريادة الأعمال · بناء قدرات الجمعيات).
3. يملأ البنود على راحته — **كل إدخال يُحفَظ تلقائياً في المتصفح** («تم الحفظ تلقائيًا»). يمكنه إغلاق الصفحة والعودة لاحقاً على نفس الجهاز ومتابعة العمل.
4. يتابع نسبة الإكمال لكل برنامج من شريط التقدّم وفي رأس كل تبويب.
5. عند الانتهاء (أو في أي وقت): يضغط **«تصدير إلى Excel»** ويرسل الملف الناتج لفريق قرارات.

## أشكال البيانات الثلاثة (Rendering rules)

- **مصفوفة سنوات** — شبكة سنوات × قيمة. خانة لكل سنة من سنوات البرنامج (افتراضياً 2022–2025؛ الفوت برنت البيئي 2026+). البنود مُجمَّعة حسب مرحلة سلسلة النتائج (مدخلات/أنشطة/مخرجات/نتائج/أثر).
- **وثيقة مرفقة** — ليست شبكة. صفّ بحالة المستند (`سأرفقها` / `مرفقة وسأرسلها` / `غير متوفرة`) + ملاحظات. **الملفات نفسها تُرسَل بالبريد بشكل منفصل** — النموذج لا يرفع ملفات.
- **قائمة مستفيدين (PII)** — **لا تُدخَل أي بيانات شخصية في النموذج إطلاقاً.** بطاقة تعليمات + حالة الجاهزية للإرسال + العدد التقريبي فقط. لا حقول لأسماء أو جوالات أو بريد. الصفحة عامة، فالقوائم تُرسَل بشكل منفصل وآمن.

## التصدير والحفظ

- **تصدير إلى Excel** — يُنشئ ملف `.xlsx` (عبر مكتبة SheetJS من CDN): ورقة لكل برنامج، تخطيط سنوات × بنود يطابق السجل، مع أعمدة حالة المستندات وحالة/عدد القوائم. الأوراق RTL بعناوين عربية. هذا هو الملف الذي يُرسله العميل.
- **الحفظ التلقائي** — كل إدخال يُحفَظ تلقائياً في متصفح الجهاز نفسه؛ أغلِق الصفحة وعُد لاحقاً على نفس الجهاز.
- **مسح الكل** — يمسح كل البيانات المحلية بعد تأكيد.

## ملاحظة الخصوصية والمستندات (مهمة)

- **المستندات** (مذكرات تفاهم، نماذج حقائب، تقارير سنوية، استبيانات) **لا تُرفَع في النموذج** — يُحدَّد وضعها فقط وتُرسَل بالبريد منفصلة.
- **قوائم المستفيدين (PII)** **لا تُدخَل في النموذج** — تُرسَل بشكل منفصل وآمن. النموذج يلتقط الحالة والعدد فقط.

## البرامج غير المُراجَعة (5 و6)

البرنامجان **التمويل المصغر** و**ريادة الأعمال** لم يُراجَعا مع العميل في الاجتماع — بُنيا من المنهجية (ن3).
بنودهما موسومة بـ **⚠ غير مُراجَع** (وسم بنفسجي على التبويب والبنود) ليراها المالك بوضوح؛ العميل يملؤها والصياغة/التوافر يُراجَعان لاحقاً.

## التبويب السابع — بناء القدرات (v4)

تبويب **بناء قدرات الجمعيات الخيرية** مبني على ملفات **v4** بانتظار تحديثات المالك.
كل المحتوى في `data.js` (لا شيء مكتوب في الـ markup) — يُحدَّث هذا التبويب لاحقاً **بتعديل `data.js` فقط** دون لمس الواجهة.

## الملفات (Source)

| الملف | الوصف |
|---|---|
| `index.html` | الواجهة (CSS مضمّن). RTL، تصميم مؤسسي أنيق. |
| `data.js` | مصدر الحقيقة الوحيد — السجل كاملاً (7 برامج → مراحل → بنود). تُعدَّل البنود من هنا فقط. |
| `app.js` | محرّك العرض + الحفظ التلقائي + شريط التقدّم + تصدير إلى Excel. |
| `build_template.py` | يولّد قالب Excel الفارغ من **نفس** `data.js` (عبر node) — يضمن تطابق النموذج والقالب. |
| `al-ahli-sijillat-template.xlsx` | قالب Excel فارغ مطابق للمخطّط (7 أوراق، RTL، عناوين عربية). |

## إعادة توليد قالب Excel

```bash
python3 build_template.py   # يتطلّب: openpyxl + node
```

## النشر (GitHub Pages)

```bash
git init && git add -A && git commit -m "al-ahli sijillat form"
gh repo create baraab-bit/al-ahli-sijillat --public --source . --push
gh api -X POST repos/baraab-bit/al-ahli-sijillat/pages -f 'source[branch]=main' -f 'source[path]=/'
# الرابط: https://baraab-bit.github.io/al-ahli-sijillat/
```

> **مهم — إبطال الكاش:** قبل كل نشر يغيّر `data.js` أو `app.js`، زِد رمز الإصدار `?v=` في `index.html` (مثال: `?v=20260624-1` ← `?v=20260624-2`). يخدم GitHub Pages هذه الملفات بـ `max-age=600`، فبدون زيادة الرمز قد يظل بناء قديم ظاهراً ~10 دقائق.
